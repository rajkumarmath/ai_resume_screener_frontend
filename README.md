## üí° What It Does & How It Works

Traditional recruitment involves manually reading hundreds of resumes to find candidates who match a specific job description. The **AI Resume Screener** automates this process using artificial intelligence, turning hours of manual screening into a few seconds of computation.

Here is the core workflow a recruiter or hiring manager takes within the app:

1. **Create a Job Context:** The user navigates to the **Jobs** dashboard and creates a new job posting. They define the Title, a detailed Description, and a list of mandatory Skills (e.g., `React, TypeScript, FastAPI`).
2. **Upload Applicant Data:** In the **Resumes** dashboard, the user uploads PDF or Word document resumes received from applicants, tagging them with the candidate's name.
3. **Run the AI Matcher:** On the **AI Match** page, the user selects a specific Job from a dropdown menu and hits "Run Match". 
4. **Actionable Insights:** The backend AI processes the text of the resumes against the context of the job description. The frontend then dynamically renders the results:
   * **Percentage Scores:** Candidates are ranked from highest to lowest match, visualized with animated progress bars.
   * **Smart Categorization:** Scores are color-coded into visual badges (`Top Match` for >80%, `Medium` for 50-79%, and `Low` for <50%).
   * **AI Explanations:** Instead of just giving a blind score, the app displays a human-readable explanation generated by the AI detailing *why* the candidate is a good fit (e.g., "The candidate has 3 years of React experience but lacks FastAPI").

Open http://localhost:3000 in your browser to view the application.

üì° API Integration Details
This frontend strictly adheres to the provided backend OpenAPI contracts. Notable integrations include:

OAuth2 Login: Handles application/x-www-form-urlencoded payload mapping email to the required username field.

Interceptors: Automatically attaches Authorization: Bearer <token> to all outgoing requests and redirects to /login if a 401 Unauthorized response is intercepted.

Error Handling: Gracefully catches FastAPI 422 Validation Error arrays and extracts the readable .msg string for UI display.
